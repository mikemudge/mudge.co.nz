Cloud Migration

With a company goal to migrate our data center to AWS, I led the migration of Sailthru's legacy php applications, consisting of 6 web services, ~20 processes (E.g queue consumers) and around 90 cronjobs. Dependencies such as Mongo DB, Active MQ and Kafka were also migrated as part of this effort.
Key results I delivered include:
  • Upgraded PHP to a supported version (Php 5 -> 7)
  • Introduced new CI pipeline, dependency management (composer) and staging environment
  • Containerized applications so they could be run with docker and EKS.
  • Refactored stateful applications to use s3 to be able to be run in a stateless manner
  • Parallelized deployments to reduce duration from ~40m to 5m
  • Improved logging, monitoring and alerts to ensure application health throughout migration
  • Zero downtime migration to kubernetes (EKS)
